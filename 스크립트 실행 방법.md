# 스크립트 실행 방법

VM 초기 설정 순서대로 실행합니다.

---

## 2. 저장소 클론 (2-깃 클론.md)

```bash
git clone git@github.com:JanbBongJun/redis-infra.git ~/redis-infra
```

- 이미 클론되어 있으면 `git -C ~/redis-infra pull`로 최신화

---

## 3. Docker 설치 (3-install-docker.sh)

```bash
cd ~/redis-infra
chmod +x 3-install-docker.sh
sudo ./3-install-docker.sh
```

- Docker Engine, Docker Compose 플러그인, 공유 네트워크(`jumsim_network`) 자동 설정
- 실행 후 docker 그룹 적용을 위해 터미널 재접속 또는 `newgrp docker` 실행 필요

---

## 4. Redis .env 설정 (4-env설정.md)

```bash
cat > ~/redis-infra/redis/.env << 'EOF'
REDIS_PASSWORD=여기에_비밀번호_입력
EOF
```

- bull-board도 같은 파일(`../redis/.env`)을 참조하므로 별도 설정 불필요
- `.gitignore`에 포함되어 있어 GitHub에 올라가지 않음

---

## 5. UFW 방화벽 설정 (5-ufw-install.sh)

```bash
chmod +x 5-ufw-install.sh
sudo ./5-ufw-install.sh
```

- 기본 정책: 들어오는 트래픽 차단, 나가는 트래픽 허용
- SSH(22): 모든 곳에서 허용
- Redis(6379), Bull-Board(3000): 내부망(`192.168.45.0/24`)에서만 허용
- Docker가 UFW를 우회하는 문제 방지 규칙 자동 적용

---

## 6. Redis 실행

```bash
cd ~/redis-infra/redis
docker compose up -d
```

- 첫 실행 시 SSL 인증서 자동 생성 후 Redis 시작
- `docker logs redis`로 정상 실행 확인

---

## 7. Bull-Board 실행

```bash
cd ~/redis-infra/bullmq
docker compose up -d
```

- Redis가 먼저 실행된 상태에서 실행
- `http://192.168.45.64:3000`으로 접속 확인
