# 백엔드 서버에 Redis CA 인증서 가져오기

백엔드 서버(192.168.45.68)에서 Redis에 TLS로 접속하려면 `redis-ca.pem` 파일이 필요합니다.

## 1. Redis VM에서 인증서 복사

백엔드 서버에서 아래 명령 실행:

```bash
# 인증서 저장 디렉토리 생성
sudo mkdir -p /etc/redis/ssl

# Redis VM(192.168.45.64)에서 CA 인증서 복사
scp sysadmin@192.168.45.64:~/redis-infra/redis/ssl/redis-ca.pem certs/redis-ca.pem

# 권한 설정
sudo chmod 644 /etc/redis/ssl/redis-ca.pem
```

## 2. 필요한 파일

| 파일             | 용도                       | 필요한 곳                |
| ---------------- | -------------------------- | ------------------------ |
| `redis-ca.pem`   | CA 인증서 (서버 신뢰 검증) | 백엔드 서버 (클라이언트) |
| `redis-cert.pem` | 서버 인증서                | Redis VM만 (서버)        |
| `redis-key.pem`  | 서버 개인 키               | Redis VM만 (서버)        |

> 백엔드 서버에는 **`redis-ca.pem`만** 있으면 됩니다. 서버 인증서와 개인 키는 복사하지 마세요.

## 3. 백엔드에서 연결 테스트

```bash
# openssl로 TLS 연결 테스트
openssl s_client -connect 192.168.45.64:6379 -CAfile /etc/redis/ssl/redis-ca.pem
```

## 4. Node.js(NestJS/BullMQ) 연결 예시

```typescript
import { readFileSync } from "fs";

const redisOptions = {
  host: "192.168.45.64",
  port: 6379,
  password: process.env.REDIS_PASSWORD,
  tls: {
    ca: readFileSync("/etc/redis/ssl/redis-ca.pem"),
  },
};
```
